TIPOS DE ATAQUES DoS/DDoS

Um guia completo de resposta a incidentes de Negação de Serviço Distribuído. Guia de como evitar diferentes tipos de ataques DoS/DDoS com estratégias de mitigação específicas.

1

Amplificação/Reflexão

Usam servidores legítimos da internet para multiplicar o ataque. Um pequeno pedido gera uma resposta gigante contra a vítima. Alto impacto com baixo esforço.

Criticidade: Crítica
2

Volumétricos Diretos

O único objetivo é enviar o máximo de lixo digital possível até que a conexão de internet da vítima fique totalmente entupida. Ataque de força bruta que satura a banda disponível. Simples mas efetivo.

Criticidade: Alta
3

Exploração de Protocolos

Aproveitam falhas nas regras básicas de comunicação. Travam servidores explorando handshakes incompletos e tabelas de estado.

Criticidade: Média
4

Camada de Aplicação

Atacam diretamente sites/sistemas, fingindo ser acessos legítimos. Baixo volume mas alto impacto no processamento.

Criticidade: Crítica
5

Exaustão de Recursos

Mira em partes específicas e "caras" do sistema: banco de dados, SSL, APIs. Consomem CPU e memória de forma seletiva.

Criticidade: Alta
6

Camada 2 (Data Link)

Acontecem dentro da rede local. Atacam switches, roteadores e protocolos internos. Isolado mas devastador internamente.

Criticidade: Média
7

Multi-Vetoriais

Combinação de várias técnicas simultaneamente. Evolutivo e adaptativo. O mais difícil de mitigar atualmente.

Criticidade: Crítica

Amplificação/Reflexão

Criticidade: Crítica

Comprometimento

O atacante utiliza servidores vulneráveis (como DNS ou NTP) para enviar respostas volumosas ao endereço IP da UFRGS, congestionando o link de internet e tirando do ar portais como o Portal do Aluno ou o SEI.

Como Funciona:

  • DNS Amplification: Amplificação 28-54x usando queries "ANY"
  • NTP Amplification: Comando monlist (até 556x de amplificação)
  • Memcached: Até 51.000x - o mais potente já registrado
  • SSDP/SNMP/CLDAP: Outros protocolos vulneráveis

Como Evitar

  • Implementar filtragem de tráfego de entrada
  • Garantir que os servidores internos não permitam consultas recursivas abertas para fontes externas
  • Implementar BCP38/Ingress Filtering na borda da rede
  • Rate limiting para protocolos vulneráveis (DNS, NTP)
  • Configurar servidores para não responder a queries externas

Como Mitigar

  • Utilizar serviços de limpeza de tráfego (scrubbing centers)
  • Utilizar redes Anycast para dispersar e absorver o volume excessivo
  • Utilizar serviços de scrubbing DDoS especializados
  • Monitorar tráfego assimétrico (muito inbound, pouco outbound)
  • Verificar se servidores internos não estão sendo usados como amplificadores
×

Volumétricos Diretos

Criticidade: Alta

Comprometimento

Inundações massivas de pacotes (UDP ou ICMP) que visam saturar completamente a largura de banda da rede da universidade, isolando-a da internet.

Tipos Principais:

  • UDP Flood: Pacotes UDP para portas aleatórias
  • ICMP Flood (Ping Flood): Respostas massivas de ping
  • GRE/ESP Flood: Protocolos de tunelamento menos monitorados
  • Ataques de Requisição-Resposta: Muitas requisições legítimas simultâneas

Como Evitar

  • Manter largura de banda sobressalente
  • Utilizar soluções de proteção DDoS baseadas em nuvem que aguentem ataques na escala de Terabits
  • Contratar capacidade de banda superior ao ataque esperado
  • Utilizar Anycast DNS para distribuir o tráfego
  • Implementar CDN para conteúdo estático

Como Mitigar

  • Bloquear tráfego de origens conhecidas por botnets
  • Aplicar regras de descarte (drop) em roteadores de borda para protocolos não essenciais
  • Acionar provedor de scrubbing DDoS
  • Configurar rate limiting agressivo na borda
  • Blackhole routing para tráfego malicioso identificado

Exploração de Protocolos/Estado

Criticidade: Média

Comprometimento

Exploram fragilidades em protocolos de rede (como o TCP SYN Flood), consumindo as tabelas de estado de firewalls e balanceadores de carga da UFRGS, impedindo novas conexões legítimas.

Técnicas Comuns:

  • SYN Flood: Envia pacotes SYN sem completar o handshake
  • ACK/PUSH ACK Flood: Variações do ataque TCP state-exhaustion
  • TCP Connection Exhaustion: Estabelece muitas conexões legítimas
  • IP/ICMP Fragmentation: Pacotes fragmentados para sobrecarregar reassembly

Como Evitar

  • Configurar adequadamente o endurecimento (hardening) de dispositivos de borda e firewalls para gerenciar melhor as filas de conexão
  • Habilitar SYN cookies nos servidores e firewalls
  • Ajustar timeouts TCP para valores mais curtos
  • Implementar State Table Protection
  • Configurar firewalls stateful com inspeção profunda

Como Mitigar

  • Implementar tecnologias como SYN cookies
  • Reduzir tempos de timeout para conexões pendentes
  • Limitar número máximo de conexões por IP
  • Usar IPS/IDS configurados adequadamente

Camada de Aplicação (Layer 7)

Criticidade: Crítica

Comprometimento

Visam aplicações específicas (HTTP/HTTPS), simulando tráfego legítimo para exaurir recursos do servidor web. Isso pode paralisar o Moodle durante períodos de alta demanda, como semanas de provas.

Técnicas Avançadas:

  • HTTP/HTTPS Flood: Requisições massivas para páginas dinâmicas
  • Slowloris: Mantém conexões abertas com headers incompletos
  • RUDY (R-U-Dead-Yet): Submissão lenta de formulários POST
  • Cache Bypass: Força requisições com parâmetros únicos para evitar cache

Como Evitar

  • Utilizar um WAF (Web Application Firewall) para filtrar solicitações suspeitas
  • Validar se o tráfego é de usuários humanos ou bots
  • Implementar WAF (Web Application Firewall)
  • Rate limiting por IP, sessão e URL
  • Utilizar CAPTCHAs e desafios para comportamento suspeito

Como Mitigar

  • Aplicar rate limiting (limite de requisições por IP)
  • Usar inspeção comportamental para identificar anomalias em tempo real
  • Análise comportamental de usuários
  • Cache agressivo de conteúdo estático
  • Implementar CDN com proteção DDoS
×

Exaustão de Recursos Específicos

Criticidade: Alta

Comprometimento

Focam em componentes específicos, como ciclos de CPU, memória ou bancos de dados de sistemas de pesquisa, tornando-os lentos ou inoperantes sem necessariamente inundar a rede.

Alvos Comuns:

  • SSL/TLS Renegotiation: Força renegociação constante de handshakes
  • XML/SOAP Attacks: Documentos XML malformados ou profundamente aninhados
  • Connection Pool Exhaustion: Abre todas as conexões disponíveis no pool de banco
  • WordPress Pingback: Explora funcionalidade para criar loops

Como Evitar

  • Monitorar o uso de recursos em tempo real
  • Isolar serviços críticos em máquinas virtuais ou containers distintos
  • Rate limiting por recurso específico
  • Otimizar queries e processos computacionalmente caros
  • Implementar circuit breakers para recursos sobrecarregados

Como Mitigar

  • Escalar horizontalmente os servidores (balanceamento de carga)
  • Realizar o reinício automático de serviços que atinjam limites críticos
  • Escalonar recursos específicos sob ataque
  • Limitar tamanho de uploads e complexidade de XML
  • Monitorar métricas específicas de recursos (CPU por processo)

Camada 2 (Data Link)

Criticidade: Média

Comprometimento

Ocorrem dentro da rede local (LAN) da UFRGS, como o envenenamento de ARP (ARP Spoofing), permitindo que um atacante intercepte dados acadêmicos ou administrativos sensíveis.

Técnicas de Camada 2:

  • ARP Poisoning/Flooding: Envia respostas ARP falsas
  • MAC Flooding: Enche a CAM table do switch com MACs falsos
  • VLAN Hopping: Explora configurações incorretas de trunking
  • STP Attacks: Interfere no Spanning Tree Protocol

Como Evitar

  • Ativar recursos de segurança em switches, como Port Security
  • Configurar Dynamic ARP Inspection
  • Implementar DHCP Snooping
  • Implementar Port Security em switches
  • Configurar DHCP Snooping
  • Usar Dynamic ARP Inspection (DAI)

Como Mitigar

  • Segmentar a rede em VLANs
  • Utilizar autenticação baseada em certificados (802.1X) para controle de acesso
  • Implementar 802.1X para autenticação de rede
  • Configurar VLANs corretamente (não usar VLAN 1)
  • Monitorar tráfego broadcast excessivo

Ataques Multi-Vetoriais

Criticidade: Crítica

Comprometimento

Combinam várias técnicas (ex: um ataque volumétrico para distrair a equipe enquanto um ataque de Camada 7 rouba dados), tornando a defesa extremamente complexa.

Estratégia Típica:

  • Fase 1: Ataque volumétrico para forçar tráfego para scrubbing centers
  • Fase 2: Ataque de protocolo para sobrecarregar dispositivos de scrubbing
  • Fase 3: Tráfego Layer 7 que parece limpo após scrubbing
  • Fase 4: Ataque adaptativo baseado nas defesas detectadas

Como Evitar

  • Adotar uma estratégia de Defesa em Profundidade, com múltiplas camadas de proteção trabalhando de forma integrada
  • Plano de resposta integrado para múltiplas equipes
  • Combinação de soluções (WAF + Scrubbing + Análise Comportamental)
  • Resposta adaptativa em tempo real com automação

Como Mitigar

  • Utilizar plataformas de orquestração de segurança (SIEM e SOAR) que automatizam a resposta a incidentes complexos em segundos
  • Monitoramento holístico de todas as camadas
  • Testes regulares de resposta a incidentes complexos
  • Parceria com múltiplos provedores de mitigação
